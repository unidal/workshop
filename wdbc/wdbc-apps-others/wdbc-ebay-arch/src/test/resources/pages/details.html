  • / ► Reflecting Changes on the Web site Without Delay
 

Generally, when the load or traffic is low, a Web site is much faster and the changes are reflected on a Web site immediately. But, when the traffic is too high, the performance of the site tends to degrade. A good Web site design should ensure that changes made by the user are reflected immediately on the Web site.

Latency is the minimum time taken between requesting for data and retrieving it. There are two types of latency:

  • User Experience Latency: It is the time taken to get the next page back to the user . It is decided by the site speed and processing time.
  • Data or Execution Latency: The user performs an operation on the site. The time taken to reflect that on relevant pages of the site is data or execution latency.

The ebay.com Web site is accessed by millions of users across a wide geography. Execution latency affects the user experience. When a user makes a change on one page, they expect that change to be reflected immediately on another page. For example, when a user sells an item, that sale should appear quickly on the My eBay page.

eBay uses Shareplex, a third party database replication software to cut down the Execution latency time. Shareplex latency is defined as the time it takes for the changes to appear and be available on the target system after it is made on the primary database.
